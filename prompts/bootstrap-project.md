# Bootstrap a Project Repository

Canonical source: `capcutfor1month-oss/project-Pipline`

1. Read `BOOTSTRAP_CONTRACT.md`, `MANIFEST.md`, `docs/PIPELINE.md`, and `docs/TOOLING.md`.
2. Inspect the target repository before writing.
3. Classify it as empty, starter, or existing product.
4. Preserve existing code and project-specific documents.
5. Detect files with the same purpose as pipeline files.
6. Create or merge only missing prerequisites.
7. Use a setup branch for an existing repository.
8. Record all planned tools and truthful statuses.
9. Do not begin product discovery, architecture, or implementation without separate approval.
10. Run pipeline validation and report evidence.

Final report:

- Repository classification
- Existing files preserved
- Files created
- Files updated
- Conflicts or duplicates found
- Validation results
- Visibility warning
- Exact next action
