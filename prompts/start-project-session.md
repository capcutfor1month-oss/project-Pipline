# Start a Project Session from a GitHub URL

Input: one target GitHub repository URL.

Canonical common pipeline:

`https://github.com/capcutfor1month-oss/project-Pipline`

## Instructions

1. Read `START_HERE.md` from the common pipeline repository.
2. Read the common bootstrap contract, manifest, pipeline, tooling registry, and agent rules.
3. Inspect the target repository before responding.
4. Recover current state from canonical project documents and active change files.
5. Detect whether the common pipeline is missing, partial, or installed.
6. Do not ask the founder to repeat the common workflow.
7. Do not rely on old chat memory when GitHub contains newer project truth.
8. Do not code, select architecture, or make product decisions unless explicitly requested.
9. Return the standard project-session status summary.
10. Continue from the repository's recorded next action or the founder's new task.
